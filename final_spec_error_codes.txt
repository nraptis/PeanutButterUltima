Final Error Code Specification
==============================

Scope
- This file describes the current error codes surfaced by archive, unarchive, and recover operations.
- Encryption remains a black box in this document.

Return surface
- `OperationResult` exposes:
  - `mSucceeded`
  - `mCanceled`
  - `mErrorCode`
  - `mFailureMessage`

Code table

0  `NONE`
- Success.

1  `CANCELED`
- The user canceled the operation.
- Bundle, unbundle, and recover all use boundary-aware cancellation.

2  `INVALID_REQUEST`
- Inputs were invalid before useful work could proceed.
- Common causes:
  - empty source or destination path
  - invalid archive block count
  - no usable source entries for bundle
  - no usable archive family for decode

3  `FILE_SYSTEM`
- File or directory I/O failed.
- Common causes:
  - source file could not be opened
  - archive file could not be read
  - output file could not be written
  - destination or parent directory could not be created

4  `CRYPT`
- Seal (encrypt) or unseal (decrypt) failed.
- In strict unbundle this is terminal.
- In recover, per-block unseal failures are tolerated and skipped; `CRYPT` is still used for setup-time failures such as failing to create the unseal engine.

5  `ARCHIVE_HEADER`
- Archive or recovery structure was not acceptable for strict decode.
- Common causes:
  - no readable encryption strength could be resolved from the selected archive family
  - archive too short for the requested block
  - recovery header could not be parsed
  - strict decode ran out of archive space before the terminator

6  `GAP_001`
- Strict unbundle encountered a missing archive slot in the selected dense index window.
- Recover logs and skips those gaps instead of surfacing `GAP_001` as the final result.

7  `BLOCK_CHECKSUM`
- Strict unbundle found a checksum mismatch in one L3 block.
- Recover counts the block as failed and continues instead of surfacing this code.

8  `RECORD_PARSE`
- Strict unbundle could not parse the logical record stream.
- Common causes:
  - path length exceeds `2048`
  - unsafe relative path
  - implausible content length
  - failed output-file creation for a decoded record
  - failed output-file write during decode

9  `RECOVER_EXHAUSTED`
- Recover consumed the available archive span without reaching a terminator and without recovering any files.

255 `INTERNAL`
- An internal invariant or format-math assumption failed.
- Common causes:
  - impossible serialization state
  - overflow during archive planning
  - invalid payload math under assumed constants

Mode notes
- Strict unbundle is fail-fast:
  - gaps, short blocks, unseal failures, checksum failures, and record-parse failures stop the run immediately.
- Recover is best-effort:
  - many per-block failures are logged and skipped
  - the final result is often `NONE` even when some blocks were lost
- Cancellation is explicit:
  - `mErrorCode = CANCELED`
  - `mCanceled = true`

String mapping
- Current names emitted by `ErrorCodeToString(...)`:
  - `NONE`
  - `CANCELED`
  - `INVALID_REQUEST`
  - `FILE_SYSTEM`
  - `CRYPT`
  - `ARCHIVE_HEADER`
  - `GAP_001`
  - `BLOCK_CHECKSUM`
  - `RECORD_PARSE`
  - `RECOVER_EXHAUSTED`
  - `INTERNAL`
