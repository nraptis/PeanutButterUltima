Final File Format Specification
===============================

Scope
- Defines PBTR archive binary format for the current engine version.
- Legacy versions are not supported by current parser checks.

Versioning and constants
- Header magic: `0xDECAFBAD`
- Footer magic constant exists in code (`0xF01DAB1E`) but is not serialized in archive files.
- Version required at decode: major `1`, minor `2`
- `ARCHIVE_HEADER_LENGTH = 40` bytes
- `RECOVERY_HEADER_LENGTH = 40` bytes
- `MAX_ARCHIVE_COUNT = 65535`
- `MAX_BLOCKS_PER_ARCHIVE = 2048`
- Release build default:
  - `BLOCK_SIZE_L3 = 144040`
  - payload bytes per block = `144000`

Byte order
- All integer fields are little-endian.

Archive file layout
1. Archive header (40 bytes)
2. `N` fixed-size L3 blocks (each `kBlockSizeL3` bytes)

Archive header layout (40 bytes)
Offset  Size  Field
0       4     `mMagic` (`0xDECAFBAD`)
4       2     `mVersionMajor`
6       2     `mVersionMinor`
8       4     `mArchiveIndex`
12      4     `mArchiveCount`
16      4     `mPayloadLength` (bytes in this archive payload)
20      1     `mRecordCountMod256`
21      1     `mFolderCountMod256`
22      2     `mReserved16`
24      8     `mReservedA`
32      8     `mReservedB`

L3 block layout
1. Recovery header (40 bytes)
2. Payload (`kPayloadBytesPerL3` bytes)

Recovery header layout (40 bytes)
Offset  Size  Field
0       8     `checksum.word1`
8       8     `checksum.word2`
16      8     `checksum.word3`
24      8     `checksum.word4`
32      2     `skip.archive_distance` (u16)
34      2     `skip.block_distance` (u16)
36      4     `skip.byte_distance` (u32)

SkipRecord semantics
- Encodes forward logical distance to next record boundary in payload space.
- Interpreted relative to current block position using known `blocks_per_archive`.
- Zero-distance skip is considered non-actionable for stride-jump recovery logic.

Checksum semantics
- Checksum is computed from:
  1) payload bytes only (not header bytes), and
  2) entire `SkipRecord` (all 8 skip bytes) with per-word tag mixing.
- Four 64-bit checksum words are stored.
- Decode requires exact checksum match to accept block as healthy.

Logical record grammar (payload stream)
- `u16 path_length`
- `path_length` bytes of path (relative path string bytes)
- `u64 content_length`
- `content_length` bytes of file content (if file record)

Special values
- `path_length == 0` => logical terminator record
- `content_length == 0xFFFFFFFFFFFFFFFF` => directory record marker (no content bytes)

Path and content constraints
- Path length must be `1..kMaxPathLength` for non-terminator records.
- Paths must be safe relative paths (no absolute prefixes, no `.`/`..`, no control bytes).
- Content length above parser plausibility threshold (`1 << 40`) is rejected.

Archive naming format
- Name writer: `prefix + source_stem + "_" + zero_padded_ordinal + suffix`
- Zero padding width equals digits in `(archive_count - 1)`.
- Parser extracts template from trailing numeric run in basename.

Discovery model for decode/recover
- Input is a filename list; parser groups by anchor template (prefix+suffix).
- Sorted numeric indices define sequence.
- Missing indices become empty `ArchiveFileBox` entries.
- Empty boxes are fatal in unbundle and recoverable damage in recover.

Corruption and gap behavior at format level
- Header parse failure, short reads, checksum mismatch, and parse errors all represent malformed/corrupt archive content.
- Recover mode may continue across such regions; strict unbundle does not.
