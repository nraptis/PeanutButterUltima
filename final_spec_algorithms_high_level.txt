Final Algorithm Specification (High Level)
==========================================

Scope
- Describes active high-level behavior for Bundle, BundleWithMutations, Unbundle, Recover, and Validate.
- This is intentionally medium-detail and implementation-aligned.

Global principles
- Streaming I/O only (no whole-file payload loads).
- Deterministic archive planning from sorted source entries.
- Cooperative cancellation in long-running loops.
- Progress logging by processed bytes and/or processed items.
- Error outcomes are trap-able via stable error codes.

Bundle
1. Validate request and archive block count.
2. Discovery phase:
  - sort source entries by relative path
  - resolve file lengths
  - build logical record stream length
  - compute archive plan (count, names, record/folder counters)
3. Flight phase:
  - for each archive, write archive header
  - for each block, stream payload bytes from logical records
  - compute recovery skip + checksum
  - write recovery header
  - optionally encrypt block
  - append block to archive
4. Emit summary and return success.

BundleWithMutations
1. Apply file mutations (create/delete entries).
2. Run discovery on mutated entry set.
3. Validate data mutation ranges.
4. Bundle flight:
  - stream payload bytes
  - apply data mutations to payload before checksum
  - write checksum/skip/encryption exactly like normal bundle

Unbundle (strict decode)
1. Validate request and destination.
2. Discovery by filename template + numeric index selection.
3. Build archive box list (including empty boxes for missing indices).
4. Decode block-by-block in strict mode:
  - read/decrypt block
  - validate checksum
  - parse records and write files/folders
5. Stop only at logical terminator or terminal error.

Recover (best effort decode)
1. Same discovery pipeline as unbundle.
2. Decode block-by-block in recover mode:
  - tolerate missing/corrupt blocks
  - reset parser on damage
  - count failed blocks
  - on parse failures, optionally jump via validated recovery stride
3. Return success if terminator is reached or if files were recovered before exhaustion.
4. Emit recover summary (files/bytes/failed blocks/time/output path).

Validate
1. Validate left/right directories.
2. Discovery logging:
  - enumerate files and directories recursively for both sides
  - report scanned item counts and discovered file counts
3. Compare phase:
  - match by relative path
  - compare bytes in 64KB chunks
  - classify hidden vs normal mismatches
4. Emit capped mismatch detail lines and summary.

Cancellation model
- Cancel checks are placed in discovery, archive loops, and parse/write loops.
- If currently inside a file write/read unit, operation attempts to finish the current file within grace period.
- On boundary/timeout, operation exits with `CANCELED`.
