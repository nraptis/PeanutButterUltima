Unpack/Recover Failure Semantics (Simplified)

DEPRECATION DECREE
- Legacy fine-grained integer failures are deprecated.
- Runtime objective is Safety + Fence behavior over exact per-variant code accuracy.

Core grouped failures

1. `UNP_RHD_FENCE` (Recovery header is invalid)
- outside file fence
- inside recovery header fence
- inside archive header fence
- recover flow: outside selected-file fence

2. `UNP_FNL_FENCE` (Path length is invalid)
- `path_length == 0`
- `path_length > MAX_VALID_FILE_PATH_LENGTH`
- outside file fence
- inside recovery header fence
- inside archive header fence

3. `UNP_FDL_FENCE` (File contents length is invalid)
- `file_contents_length > MAX_VALID_FILE_CONTENT_LENGTH`
- outside file fence
- inside recovery header fence
- inside archive header fence

Fence/Safety runtime rules

- Any out-of-range, overflow, or non-progress condition must hard-fail immediately.
- Failures must happen before dangerous allocation or large file writes.
- Prefer broad grouped failure classification over continued parsing.

EOF grouped behavior

- `UNP_EOF_001`: dangling archives remain after logical stop marker.
- `UNP_EOF_002`: dangling bytes remain in current archive after logical stop marker.
- Precedence: `UNP_EOF_001` over `UNP_EOF_002`.

Failure bubbling rule

- Candidate-set failures do not fail globally until all candidates fail.
- Recover may try multiple candidates; stop on first success.
- If all candidates fail, report the first meaningful Safety/Fence failure.
