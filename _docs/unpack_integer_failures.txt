Unpack Integer Failure Semantics

Purpose
- Capture invalid integer cases during unpack.
- Every case below should:
  - trigger a hard unpack failure
  - generate a log entry

General notes
- "REMAINING_BYTES_IN_UNPACK_JOB" means remaining logical unpack bytes from the current read point to the end of the unpack stream.
- Using a big integer for comparison is acceptable if that makes overflow-safe comparisons easier.
- "lands inside of a recovery header" means the decoded target offset resolves to bytes that belong to a recovery header span, not payload.
- "lands inside of an archive header" means the decoded target offset resolves to bytes in the 36-byte archive header region.


File name length integer failures

UNP_FNL_001 file_name_length > MAX_VALID_FILE_PATH_LENGTH
  Result:
  - fail unpack
  - log error

UNP_FNL_002 file_name_length > REMAINING_BYTES_IN_UNPACK_JOB
  Result:
  - fail unpack
  - log error

UNP_FNL_003 file_name_length == 0
  Result:
  - fail unpack
  - log error

UNP_FNL_004 file_name_length lands inside of a recovery header
  Meaning:
  - the bytes needed to read the file name length cross into or begin inside a recovery header region
  Result:
  - fail unpack
  - log error

UNP_FNL_005 file_name_length lands inside of an archive header
  Meaning:
  - the bytes needed to read the file name length cross into or begin inside an archive header region
  Result:
  - fail unpack
  - log error

Recovery job special first-header flow

- In the recovery job, the user-selected archive has a special first recovery header flow.
- For that flow, `UNP_RHD_001`, `UNP_RHD_002`, `UNP_RHD_003`, and `UNP_RHD_004` still apply.
- The fact that the archive was user-selected does not relax those checks.

RCV_RHD_001 recovery-job special first-header flow:
    non_first_recovery_next_file_distance lands outside of this archive
  Meaning:
  - the resolved next-file position is not within the payload region of the selected archive
  - this is stricter than REMAINING_BYTES_IN_UNPACK_JOB because the recovery-start rule is anchored to the selected archive
  Result:
  - fail recovery
  - log error


Implementation rule

- Do not silently normalize or skip any of the above.
- Each invalid integer case is corrupt input.
- Corrupt input must stop unpack immediately and emit a log.
