Archive Header Rules

Definitions

- Archive header:
  - the 36-byte clear-text header at the start of each archive
  - archive headers are not part of payload
  - every archive has exactly one archive header

- Payload:
  - all bytes after the 36-byte archive header
  - payload includes recovery headers
  - payload includes serialized file-record bytes

Archive header layout

- Bytes `0..3`
  - magic header
  - little-endian `0xF01DAB1E`

- Byte `4`
  - recovery-enabled flag
  - `1` means this archive is marked recoverable
  - `0` means this archive is not marked recoverable

- Byte `5`
  - reserved
  - must be `0`

- Bytes `6..9`
  - major version
  - little-endian uint32

- Bytes `10..13`
  - minor version
  - little-endian uint32

- Bytes `14..19`
  - archive sequence number
  - little-endian uint48
  - archive sequence `0` is the first archive in the set

- Bytes `20..27`
  - archive identifier
  - 8 raw bytes
  - all archives in the same set must share the same archive identifier

- Bytes `28..31`
  - reserved archive-header bytes
  - must be `0`

- Bytes `32..35`
  - magic footer
  - little-endian `0xDECAFBAD`

Behavioral rules

- Archive headers are always clear-text.
- Archive headers are never encrypted.
- Archive headers are always stored before payload.
- Archive-header bytes are never counted as payload bytes.
- Archive selection and grouping may rely on:
  - archive identifier
  - archive sequence number
  - recovery-enabled flag

Comparison rules

- Two archives are header-equal when:
  - all header bytes match exactly
- Two archive sets are identifier-equal when:
  - each archive in the set carries the same archive identifier bytes
- Recovery-header randomization does not affect archive-header equality, because recovery headers are payload, not archive header.

Known error-code categories

- `AHD`
  - archive-header semantic failures should be categorized under archive-header codes
- `SYS`
  - system/read/open failures surrounding archive-header handling may use system codes until more specific registry entries are added

Future work

- Add explicit archive-header failure codes once the corrupt-header cases are enumerated in the same way as unpack integer failures.
