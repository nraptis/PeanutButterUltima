Findings fixed in this pass
- Fixed generated folder cases dropping empty directories in C++ emission:
  - Added `mInputEmptyDirs` to generated case struct.
  - Propagated into `FenceRoundTripSpec.mOriginalEmptyDirectories`.
  - Bundle stage now materializes empty dirs from seed spec.
- Fixed round-trip harness over-constraining unbundle fence cases:
  - Added `mRunRecoverAfterMutation` and `mRequireRecoverTreeMatch`.
  - Generated unbundle-flow fence tests now do not fail due recover tree mismatch.
- Fixed zsh test log summary parsing and layout in run_brew_tests.sh:
  - Failing tests listed first.
  - One-line parsed failure summary per failed test.
  - Full logs grouped by failed test.
- Fixed generator recovery-header offset model for truncated archives:
  - Recovery-header selection now uses actually present blocks, not theoretical max blocks.
  - Illegal target distance chooser now uses actual archive byte lengths (no fixed-size tail assumptions).
- Fixed generator requiring at least one file in all cases:
  - Prevents invalid spec with no source files.
- Fixed generator stream model drift for directory records:
  - Added 6-byte directory content marker (`0xFFFFFFFFFFFF`) in logical stream model.
  - Updated zero-marker offset accounting accordingly.
- Hardened EOF case anchor selection:
  - Ensures selected content-length field is large enough to be mutated (non-no-op).

Remaining failure clusters
- EOF trailing tests:
  - EndOfFile_TrailingGarbage
  - EndOfFile_TrailingArchives
  - Symptom: mutated unbundle unexpectedly succeeds.
  - Likely cause: current EOF mutation heuristic is still too weak for all generated trees.
- Header-targeted length tests:
  - FileContent_InArchiveHeader / InRecoveryHeader
  - FileName_InArchiveHeader / InRecoveryHeader
  - FolderName_InArchiveHeader / InRecoveryHeader / MissingArchive / Outlying
  - Symptom: either unbundle succeeds or fails with a non-expected code path.
  - Likely causes:
    - Mutation value class can still land in a different practical failure mode than requested target class.
    - Runtime path-byte validation may fire before fence tokenized message in some branches.
- RecoveryStride tests (all 5 variants):
  - InArchiveHeader / InRecoveryHeader / MissingArchive / Outlying / Zero
  - Symptom: expected UNP_RHD_FENCE not observed; recover walk often reports UNP_EOF_003.
  - Likely cause:
    - Single recovery-header mutation is not sufficient to force recover walk into a path that evaluates the mutated stride as intended.
    - These need a two-mutation scenario (trigger recover walk + stride mutation), or a dedicated runtime mutation solver.

Conclusion
- The previous generator definitely had structural drift bugs (now fixed).
- Remaining failures are mostly semantic-case construction problems, not just syntax/output issues.
- Next reliable step is a runtime mutation-solver layer for generated cases:
  - Generate semantic test intents in Python.
  - Resolve exact mutation bytes/offsets against real bundled archives in C++ before running assertions.
  - This avoids stale layout assumptions and prevents self-fulfilling expectations.
